## File Naming Convention

Screenshots follow a numbered prefix system (01- through 17-) to maintain order and make updates easier. When replacing a screenshot:

1. Keep the same filename to avoid breaking documentation links
2. Ensure the image shows the same feature/section
3. Use PNG format for best quality
4. Capture at a reasonable resolution (1200px+ width recommended)

## Adding New Screenshots

If new features are added:

1. Add new screenshot with next number in sequence (e.g., `11-new-feature.png`)
2. Update this README.md with description
3. Add corresponding section to USERGUIDE.md
4. Update README.md features list if applicable
